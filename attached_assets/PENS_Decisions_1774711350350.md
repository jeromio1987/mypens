Running record of confirmed product decisions. Update this file whenever something is locked in. Do not reopen closed decisions without a reason.

## Confirmed Decisions

### Product philosophy
- PENS is an interpretation system, not a tracking app ✅
- Low friction is non-negotiable — no food logging ✅
- Embrace real-life behaviour (alcohol, holidays, inconsistency) ✅

### Modes
- 3 modes confirmed: Locked In 🟢 / Balanced 🟡 / Off 🔴 ✅
- Social absorbed into Balanced (a social day is still a maintained day) ✅
- Recovery absorbed into Off (rest and chaos are both deviations — Off handles both) ✅
- "Off Track" rejected — implies failure ✅

### Core input method
- 1-tap mode selection as daily entry point ✅
- 1-tap context tags (alcohol, heavy meal, travel, late night, intense training) ✅
- No calorie logging ✅
- No macro logging ✅

### Hero feature
- Weight interpretation engine is priority 1 ✅
- Phase 1: explanatory only (no predictions) ✅
- Phase 2: personalised predictions (after 3–4 weeks user data) — pending

### Wearables
- Garmin is first priority ✅
- Strava second ✅
- PENS sits above wearables — interpretation layer, not competitor ✅

### Clubroom
- Not a dashboard — narrative layer ✅
- Weekly Card is hero format ✅
- Daily Card secondary ✅
- Phase 1: private only — no social ✅
- Daily cards not shown if data is insufficient (no generic filler) ✅

### Social
- No open social feed ✅
- No follower system ✅
- No raw data comparison ✅
- Inner Circle (Phase 2): invite-only, 3–5 people max ✅
- Curated Public (Phase 3): reactions only, no comments ✅

### Phase 1 — What is NOT built
- Social feed ✅ cut
- Insight/pattern cards ✅ cut (no data yet)
- Predictive weight numbers ✅ cut (no user history yet)
- Complex food logging ✅ cut

### LLM workflow
- Claude owns product spec and decisions ✅
- GPT handles copy iteration and brainstorming ✅
- Replit handles all code and execution ✅
- This MD folder is the shared source of truth ✅

## Rejected Ideas (and why)

| Idea | Rejected because |
|------|-----------------|
| "Off Track" as mode name | Implies failure before user starts |
| Open social feed | Destroys product integrity, adds noise |
| Calorie / macro logging | Breaks low-friction principle |
| Pattern/insight cards in Phase 1 | No data to base them on — generic = trust killer |
| Predictive weight numbers at launch | Population averages misfire = distrust |
| Full social network | Wrong audience, wrong tone |
