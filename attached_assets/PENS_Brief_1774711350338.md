# PENS — Quick Brief (paste this into any LLM to get it up to speed)

## What is PENS?
PENS is a health interpretation app for normal high-functioning adults. It is NOT a tracking app. It is a system that adapts to real life — including holidays, alcohol, and inconsistency.

Core idea: instead of asking users to log everything, the system infers from context + wearables + a single daily input (mode selection).

## The 3 Modes
| Mode | Symbol | Intent |
|------|--------|--------|
| Locked In | 🟢 | Optimise |
| Balanced | 🟡 | Maintain — covers flexible and social days |
| Off | 🔴 | Chaos + recovery — no friction, reset included |

## Hero Feature
Weight interpretation engine — explains weight fluctuation (water retention, glycogen, alcohol, sodium) before the user panics. Rule-based logic, no ML needed at launch.

## Core Stack
- Input: 1-tap mode + 1-tap context tags (alcohol, heavy meal, travel, late night)
- Data: Garmin (HRV, sleep, HR, steps), Strava (activity), scale (weight)
- Output: plain-language interpretation, daily card, weekly summary card

## The Clubroom
Narrative layer of the app. Not a dashboard. Turns data into stories. Weekly Card is the hero format. Tone: masculine, intelligent, real-life, "continental collective."

## What NOT to build at launch
- Social feed, followers, comments
- Detailed food logging
- Predictive weight numbers (needs 3–4 weeks user data first)
- Insight Cards based on pattern detection (no patterns yet)

## Positioning
"Health that adapts to real life." / "No logging. Just clarity."
Target: normal high-functioning adults, not biohackers.
Reference: WHOOP × Apple Health × lifestyle magazine × anti-perfection philosophy.
